### 🔐 القواعد الذهبية التي أطبقها (12 قاعدة)

1. **Two-Task Rule** — لا أكثر من مهمتين في كل دورة
2. **Fix-Now** — أي خطأ يُصلح فوراً
3. **Full Consistency** — فحص كل الملفات المتأثرة بعد أي تعديل
4. **Explicit Command** — أوامر دقيقة للمنفّذ
5. **Preemptive Prevention** — منع الأخطاء قبل حدوثها
6. **Command Brief** — بطاقة توضيحية قبل كل أمر
7. **i18n-Only** — لا نصوص عربية في الكود، فقط مفاتيح `.ftl`
8. **Shared-First** — أي كود متكرر يُستخرج لـ `bot/utils/`
9. **AI-Ready Architecture** — البنية جاهزة للذكاء الاصطناعي
10. **Zero-Defect Gate** — لا تقدم بكود معطوب
11. **SpecKit-Only Documentation** — كل التوثيق عبر أوامر SpecKit
12. **GitNexus-Enhanced Development** — فحص Blast Radius قبل تعديل أي كود مشترك

```typescript
export async function save(ctx, data, action) {
  let attempts = 0;
  const MAX_RETRIES = 1;
  
  while (attempts <= MAX_RETRIES) {
    try {
      await action(data);
      return; // نجح ✅
    } catch (error) {
      attempts++;
      
      if (attempts > MAX_RETRIES) {
        // فشل بعد محاولتين
        await ctx.reply(
          ctx.t('module-kit-save-failed-persistent'),
          { reply_markup: { inline_keyboard: [[
            { text: ctx.t('button-retry'), callback_data: 'retry_save' }
          ]]}}
        );
        throw error;
      }
      
      // انتظر ثانية قبل المحاولة الثانية
      await new Promise(resolve => setTimeout(resolve, 1000));
    }
  }
}
```

```typescript
async function showUserProfile(ctx: BotContext, telegramId: bigint) {
  const user = await prisma.user.findUnique({
    where: { telegramId },
    include: {
      adminScopes: {
        include: { section: true }
      },
      joinRequest: true,
      _count: {
        select: {
          auditLogs: true
        }
      }
    }
  })

  if (!user) {
    return ctx.answerCallbackQuery(ctx.t('errors-user-not-found'))
  }

  const maskedNationalId = user.nationalId 
    ? `${user.nationalId.substring(0, 4)}${'*'.repeat(10)}`
    : ctx.t('value-unknown')

  const text = ctx.t('user-profile-full', {
    name: user.fullName,
    nickname: user.nickname || ctx.t('value-none'),
    phone: user.phone || ctx.t('value-unknown'),
    nationalId: maskedNationalId,
    role: ctx.t(`role-${user.role.toLowerCase()}`),
    status: user.isActive ? ctx.t('status-active') : ctx.t('status-inactive'),
    createdAt: user.createdAt.toLocaleDateString('ar-EG'),
    lastActive: user.lastActiveAt 
      ? user.lastActiveAt.toLocaleDateString('ar-EG')
      : ctx.t('value-never'),
    auditCount: user._count.auditLogs,
    scopesCount: user.adminScopes.length
  })

  const keyboard = new InlineKeyboard()
    .text(ctx.t('button-back-to-user'), `user:view:${telegramId}`)

  return ctx.editMessageText(text, {
    parse_mode: 'Markdown',
    reply_markup: keyboard
  })
}
```

```
/speckit.specify Create Feature 006: UX Critical Fixes

Based on comprehensive UX audit (UX-Audit-Report.md), implement critical user experience improvements identified as CRITICAL priority.

## Problem Statement
Current admin interface has three critical UX issues:
1. Admins cannot view complete user information (missing nickname, national ID, registration date, last activity)
2. No confirmation dialog before destructive actions (accidental user deactivation risk)
3. Button text too long for mobile screens (truncated on small devices)

## User Stories

### US1: Complete User Profile View
As a Super Admin, I want to view a user's complete profile including all database fields, so that I can make informed decisions about user management.

Acceptance Criteria:
- Given I am viewing a user's details
- When I click "View Full Profile" button
- Then I see: full name, nickname, phone, masked national ID, role, status, registration date, last active date, audit count, admin scopes count
- And national ID is masked (first 4 digits + 10 asterisks)
- And I can navigate back to user details

### US2: Confirmation for Destructive Actions
As a Super Admin, I want confirmation before deactivating a user, so that I don't accidentally block someone's access.

Acceptance Criteria:
- Given I am viewing a user's details
- When I click "Deactivate" or "Activate" button
- Then I see a confirmation dialog explaining the consequences
- And I can confirm or cancel the action
- And only after confirmation is the action executed

### US3: Mobile-Optimized Button Text
As any user on mobile device, I want button text to fit completely on screen, so that I can read all options clearly.

Acceptance Criteria:
- Given I am using the bot on iPhone SE (small screen)
- When I view any menu or screen
- Then all button text is fully visible (not truncated)
- And button text is under 16 characters in Arabic

## Functional Requirements

- FR-UX-001: System MUST provide showUserProfile() function that displays comprehensive user information including nickname, masked national ID, registration date, last active date, audit log count, and admin scopes count
- FR-UX-002: System MUST show confirmation dialog before executing user activate/deactivate actions, explaining consequences and providing confirm/cancel options
- FR-UX-003: System MUST use short button text (≤16 chars) for all navigation buttons to ensure full visibility on mobile devices

## Non-Functional Requirements

- NFR-UX-001: National ID MUST be masked showing only first 4 digits followed by 10 asterisks for privacy
- NFR-UX-002: All user-facing text MUST be in i18n files (ar.ftl, en.ftl) following i18n-Only principle
- NFR-UX-003: Button text length MUST NOT exceed 16 characters in Arabic to fit on iPhone SE screen

## Success Criteria

- SC-UX-001: Admin can view complete user profile in maximum 2 clicks from user list
- SC-UX-002: Zero accidental user deactivations (100% require confirmation)
- SC-UX-003: 100% of button text visible on iPhone SE (375px width) without truncation

## Out of Scope

- User search functionality (deferred to Phase 6)
- Pagination for user list (deferred to Phase 6)
- Navigation utility (deferred to separate feature)
- Cancel buttons in conversations (deferred to Phase 6)
```
