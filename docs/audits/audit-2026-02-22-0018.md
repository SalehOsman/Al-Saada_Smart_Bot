### 🔴 حرجة (Critical) — تُسبّب فشل Runtime

---

#### [BUG-01] الـ Prisma Client قديم ولا يعكس الـ Schema الحالي

- **الملف:** جميع الملفات التي تستخدم `prisma.user`
- **السبب:** تم تشغيل `migration` (20260220...) يُغيّر اسم `first_name`/`last_name` → `full_name` ويضيف حقل `id`، لكن لم يُشغَّل `prisma generate` بعده.
- **الأثر:** الـ TypeScript client لا يزال يعرف الـ model بالأسماء القديمة (`firstName`, `lastName`)، مما يُفشل type-checking في `start.ts` و`menu.ts` ويُنتج أخطاء runtime.
- **الإصلاح:** تشغيل `npx prisma generate` بعد كل migration.

---

#### [BUG-02] أسماء الحقول في `saveJoinRequest()` خاطئة

- **الملف:** `packages/core/src/bot/conversations/join.ts` — السطر 283
- **المشكلة:**
  ```typescript
  // الكود الحالي (خاطئ)
  userId: telegramId,  // ❌ الحقل في Schema هو: telegramId
  name: data.fullName, // ❌ الحقل في Schema هو: fullName
  ```
- **الدليل:** `prisma/schema.prisma` السطران 70-71: `telegramId BigInt` و`fullName String`.
- **الأثر:** `prisma.joinRequest.create()` يفشل بـ "Unknown argument" كل مرة يحفظ المستخدم طلبه.

---

#### [BUG-03] منطق الجنس معكوس في `extractNationalIdInfo()`

- **الملف:** `packages/core/src/bot/conversations/join.ts` — السطر 29
- **المشكلة:**
  ```typescript
  // الكود الحالي (خاطئ)
  const gender = genderCode % 2 === 0 ? 'MALE' : 'FEMALE'
  ```
- **المعيار المصري الصحيح:** الرقم الفردي (Odd) في الخانة التاسعة = أنثى ← والزوجي (Even) = ذكر.
  ```typescript
  // الصحيح
  const gender = genderCode % 2 === 0 ? 'FEMALE' : 'MALE'
  ```
- **الدليل:** تعليقات الاختبار في `join.test.ts` السطر 521-525 تُثبت ذلك.
- **الأثر:** كل طلب انضمام يُخزَّن بجنس مقلوب.

#### [BUG-05] تدفق التأكيد: InlineKeyboard لا يُستقبل بـ `waitFrom`

- **الملف:** `packages/core/src/bot/conversations/join.ts` — السطران 108-122 و 255-277
- **المشكلة:** زر "تأكيد" يُرسل `callback_query` من Telegram، لكن الكود ينتظر `ctx.message?.text`.
  ```typescript
  // شاشة التأكيد ترسل callback_data
  { text: ctx.t('button_confirm'), callback_data: 'confirm_join' }

  // لكن الـ waitFrom يستقبل فقط message.text
  const confirmation = await conversation.waitFrom(ctx.from, async (ctx) => {
    return ctx.message?.text // ← دائماً undefined عند الضغط على زر!
  })
  ```
- **الأثر:** زر "تأكيد" لا يعمل أبداً — كل طلب ينتهي بـ "إلغاء" رغم ضغط المستخدم "تأكيد".

---

#### [BUG-06] `bot.conversation()` غير موجودة في grammY API

- **الملف:** `packages/core/src/bot/index.ts` — السطر 44
- **المشكلة:**
  ```typescript
  bot.conversation('join', joinConversation) // ❌ هذا الـ method غير موجود في grammY
  ```
- **الـ API الصحيح:**
  ```typescript
  import { conversations, createConversation } from '@grammyjs/conversations'
  bot.use(conversations())
  bot.use(createConversation(joinConversation, 'join')) // ✅
  ```
- **الأثر:** البوت لا يُسجَّل الـ conversation أصلاً — `ctx.conversation.enter('join')` يفشل runtime.

---

#### [BUG-07] `menu.ts` يستعلم عن علاقات غير موجودة في الـ Schema

- **الملف:** `packages/core/src/bot/handlers/menu.ts` — السطران 23-26
- **المشكلة:**
  ```typescript
  include: {
    adminScopes: {
      include: {
        section: true, // ❌ AdminScope ليس لديها relation مباشرة لـ Section
        module: true,  // ❌ AdminScope ليس لديها relation مباشرة لـ Module
      }
    }
  }
  ```
- **الدليل:** `prisma/schema.prisma` السطران 168-184: `AdminScope` لديها فقط `scopeId: String` و`scopeType: ScopeType` — لا توجد `@relation` لـ Section أو Module.
- **الأثر:** `prisma.user.findUnique` يفشل runtime عند اختيار أي مستخدم للقائمة.

#### [BUG-10] مقارنة `INITIAL_SUPER_ADMIN_ID` غير آمنة للـ BigInt الكبيرة

- **الملف:** `packages/core/src/bot/handlers/start.ts` — السطر 34
- **المشكلة:**
  ```typescript
  env.INITIAL_SUPER_ADMIN_ID === Number(telegramId)
  ```
  `Number(BigInt)` يفقد الدقة للأرقام > 2^53. الـ Telegram IDs يمكن أن تتجاوز هذا الحد.
- **الإصلاح الآمن:**
  ```typescript
  env.INITIAL_SUPER_ADMIN_ID !== undefined
  && BigInt(env.INITIAL_SUPER_ADMIN_ID) === telegramId
  ```
